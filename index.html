RISC-V operating system with Doom support

demo: doom

overview

A custom operating system built from scratch with support for running Doom. The project demonstrates fundamental OS concepts including memory management, process scheduling, and graphics rendering on RISC-V architecture.

features

  • Custom Bootloader Loads the kernel into memory and transfers control
  • Memory Management Implements paging and virtual memory
  • Process Scheduler Round-robin scheduling with priority queues
  • Graphics Driver Framebuffer support for Doom rendering
  • VirtIO Devices Block device and RNG drivers
  • File System KTFS implementation for game data

architecture

The OS is written in C and RISC-V assembly. It runs on QEMU and handles interrupts via PLIC (Platform-Level Interrupt Controller). The Doom port interfaces directly with the framebuffer device and implements custom input handling.

building

See the Makefile in sys/ for build instructions. Requires a RISC-V cross-compiler toolchain and QEMU for testing.
