Project Overview

This is a custom operating system built from scratch with support for running Doom. The project demonstrates fundamental OS concepts including memory management, process scheduling, and graphics rendering.

Key Features

  • Custom Bootloader: Loads the kernel into memory and transfers control
  • Memory Management: Implements paging and virtual memory
  • Process Scheduler: Round-robin scheduling with priority queues
  • Graphics Driver: VGA and framebuffer support for Doom rendering
  • Keyboard & Mouse Input: PS/2 controller drivers
  • File System: Basic FAT32 implementation for game data

Technical Details

The OS is written primarily in C with assembly for low-level operations. It boots via GRUB and runs in protected mode. The Doom port required implementing a custom graphics interface and input handling system compatible with the original Doom engine.

Building and Running

See the README.md in the source tree for build instructions. You'll need a cross-compiler toolchain and QEMU or VirtualBox for testing.
